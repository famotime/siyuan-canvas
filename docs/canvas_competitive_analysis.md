# 思源笔记“无界”画布插件 (siyuan-canvas) 竞品分析与演进规划

## 一、 背景与概述

随着“无界 (siyuan-canvas)” 插件在思源笔记中实现了基础的卡片编辑、节点排版、自动保存、跨设备移动端查阅以及业内领先的“演示模式”等功能，产品的基础形态已经趋于完善。
为了更好地制定下一步的演进路线，本文档系统调研了 Obsidian 官方 Canvas 核心功能、使用最广泛的第三方 Canvas 增强插件以及典型的 Mindmap 思维导图插件，通过对标分析，提炼出我们的核心竞争优势，并对后续的功能补齐和特性演进提出建议。

## 二、 竞品调研：Obsidian Canvas 及生态

### 1. Obsidian 官方 Canvas 核心能力
Obsidian 的原生 Canvas 定位于“无限 2D 思考空间”，其核心机制包括：
- **跨媒体聚合**：不仅支持纯文本卡片，还能直接嵌入 Markdown 笔记、图片、PDF、音视频，甚至完整的 Web 网页 (iframe)。
- **连接与组织**：提供灵活的连线能力，并支持通过群组 (Group) 将相关卡片框选在一起。
- **嵌套画布**：允许在画布中嵌入另一个 `.canvas` 文件，实现复杂项目的模块化。
- **底层架构**：采用开放的 JSON Canvas 格式，数据完全本地化且与其他工具具备互操作性。它并不是独立于笔记之外的白板，而是可以和 Vault 内的双链体系深度互动的视图。

### 2. 热门第三方 Canvas 增强插件
原生的 Canvas 提供了画布的“骨架”，社区则通过各类插件丰富了“血肉”：
- **Advanced Canvas**：大而全的增强套件。支持为节点应用自定义形状、样式，支持可折叠群组 (Collapsible Groups)，甚至引入了演示模式。
- **Enhanced Canvas**：强化画布与笔记之间的语义联系。它能自动同步卡片连线关系，在底层 Markdown 文件中生成真实的双向链接或属性（Properties）。
- **ZettelFlow / Cannoli**：将画布“流程化”和“智能化”。ZettelFlow 允许用画布编排记笔记的标准步骤（如模板流转）；Cannoli 更是允许在画布上直接通过连线构建无代码的 LLM（AI）调用脚本。
- **体验补齐工具**：如 **Search in Canvas**（画布内文字搜索）、**Canvas Minimap**（大型画布全局小地图导航）、**Node Auto-Resize**（节点自适应大小）以及 **Canvas Collapse Node**（折叠收缩多余节点）。

### 3. Obsidian 典型 Mindmap (思维导图) 插件
除了自由布局的白板，用户对结构化的导图也有强诉求：
- **Markmind**：目前功能最强大的现代导图工具。除了基础的树状布局，它的杀手锏是**深度整合了 PDF 标注功能**。用户可以在 PDF 上高亮文本或框选区域，并将这些标注直接关联、提取到导图节点中，非常适合重度学术和研究场景。
- **Enhancing Mindmap / Mind Map**：更传统的解决方案，侧重于 Markdown 大纲列表与可视化思维导图之间的实时双向同步，允许在图形界面直接拖拽修改层级关系。

## 三、 当前项目 (siyuan-canvas) 竞争力分析

结合目前的开发进展（如：支持录制播放路径的演示模式、移动端双指缩放查看、冷调彩虹等自定义主题等），我们进行如下分析：

### 1. 核心优势 (Strengths)
- **无缝的数据跨端与互通**：坚持标准的 `.canvas` 协议，不仅打破了思源笔记原本没有原生画布的痛点，更打通了思源与 Obsidian 用户之间的数据壁垒。
- **极具创新的“演示模式”**：这是我们目前最具差异化的长板功能。相比于一盘散沙的图谱，我们支持对节点设定播放路径、播放时自动压暗背景并高亮聚焦节点，使得知识可以“从网状的平铺转变为线性的叙述”。
- **强大的对齐与一键排版**：提供了丰富的横/竖排、网格分布、等距对齐等快捷排版操作，布局效率甚至高于竞品的原生体验。
- **移动端的优先关怀**：虽然桌面端是主力编辑场景，但我们已经提前实现了移动端的友好展示和手势缩放，填补了许多竞品在移动端体验上的空白。

### 2. 相对劣势与待填补差距 (Weaknesses)
- **富媒体与深层嵌入的缺失**：目前的 Link 卡片仅作为跳转入口，未能像 Obsidian 那样直接在画布中渲染网页、PDF 或音视频。节点形态较为单一，主要集中在文本与静态文件上。
- **大盘导航与检索效率**：对于成百上千节点的大画布，我们还缺少 Minimap (小地图) 导航机制，也没有实现在画布视图内直接 `Ctrl+F` 检索文本高亮卡片的功能。
- **信息密度的动态控制**：无法像导图或 Advanced Canvas 那样一键折叠或展开群组/节点，当内容增多时容易引发视觉疲劳，也暂不支持节点大小随内容文字自动撑满（Auto-Resize）。
- **缺乏与思源本体数据的深度融合**：目前的连线仅存在于画布内部，画布内的连接关系未能反向赋能给思源笔记的“关系图谱”或“块引用/双链”体系。

## 四、 后续功能补齐与增强特性建议 (Roadmap)

基于上述分析，为使“无界”插件不仅成为一个兼容工具，更成为思源笔记中最强大的知识沉淀与发散枢纽，建议按以下阶段进行演进：

### 阶段一：补齐基础体验缺口（Quick Wins）
**目标**：消除在大型画布场景下的摩擦力，提升日常交互顺滑度。
- [x] **画布全局小地图 (Minimap)**：在右下角或左下角引入悬浮的小地图导航模块，支持拖拽快速定位，解决无限画布带来的迷失感。
- [x] **画布内即时搜索 (Search in Canvas)**：增加顶部工具栏的搜索框，输入关键词即可对命中节点进行边框高亮或过滤，便于快速锁定信息。
- [ ] **内容自适应与折叠收缩**：
   - 增加“根据内容自适应大小”按钮/快捷键，消除大段空白。
   - 为分组 (Group) 节点引入折叠（Collapse）属性，一键收起组内零散节点，提升信息管理的层次感。

### 阶段二：增强知识聚合能力（打造核心壁垒）
**目标**：让画布真正成为一切资料的“控制中心”，不再局限于文本。
- [ ] **富媒体与网页内联预览**：升级链接和文件节点的渲染引擎。对 B站/YouTube 链接支持 iframe 播放；对 PDF 文件支持在卡片内部分页阅读；让画布成为真正意义上的参考资料看板。
- [ ] **双向关系反写 (知识库融合)**：借鉴 Enhanced Canvas 的思路，允许用户配置：当在画布中用连线连接两个“思源文档节点”时，是否自动在对应文档中生成块链接/文档链接，让可视化的拖拉拽真实构建起你的思源双链图谱。
- [x] **思维导图快捷模式 (Mindmap Mode)**：引入针对选定节点的“发散树状布局”算法，允许通过 `Tab`（子节点）和 `Enter`（同级节点）极速创建卡片和连线，兼顾白板的自由与导图的高效输入。

### 阶段三：探索高阶与智能化应用（Future Vision）
**目标**：提升生产力上限，引入自动化与学术流支持。
- [ ] **深度学术批注联动 (对标 Markmind)**：对于在画布中打开的思源 PDF 文件，支持将其高亮批注或框选区域直接“拖拽”出来，成为画布上的独立卡片，双击直接跳转回 PDF 锚点，打造终极学术文献阅读台。
- [ ] **AI 节点与工作流引擎**：探索结合本地 LLM 或思源 AI 接口。例如，允许用户在画布中创建一个“AI 提词卡片”，将多个文档卡片连线指向它，自动生成摘要结果到输出卡片。实现直观的可视化思维链提示 (Prompt Chaining)。

### 阶段四：探索真正的差异化与护城河特性（Innovation）
**目标**：脱离“模仿者”身份，依托思源底层优势（块级粒度、本地 SQL）打造人无我有的交互形态。
- [ ] **块级粒度流转与 SQL 动态节点**：允许将具体的“块 (Block)”直接投影到画布并保持双向实时同步；支持基于 SQL 查询语句生成“动态节点簇”，使画布内容随知识库更新而自动生长。
- [ ] **悬浮图谱漫游 (Sprout 模式)**：选定节点后，一键依据思源的底层文档引用关系，在周围自动“长出”关联的文档卡片。实现从单点到关系网的全自动伸展，极大减少手动拖拽组网的成本。
- [ ] **语义缩放 (Semantic Zooming) 与无缝嵌套**：打破传统的纯视觉缩放。在鸟瞰视角仅显示标题与标签，放大时显露摘要，极度放大时无缝切入完整的块编辑模式（或平滑潜入嵌套的子画布）。
- [ ] **空间记忆宫殿 (记忆褪色机制)**：将画布打造成“记忆花园”。长久未访问的节点会随时间推移在视觉上逐渐“褪色”，用户需定期巡游并交互以重新为其“上色”，将二维空间记忆与闪卡复习机制完美融合。
- [ ] **动态知识大盘 (Dashboard Widgets)**：打破白板的静态边界，引入功能型微组件卡片（如番茄钟、打卡热力图、待办聚合），让画布升级为用户每次打开思源的“个性化起居室”。
