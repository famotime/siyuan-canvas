# 画布”无界”，思考有方 —— 一个兼容 Obsidian Canvas 的思源笔记插件

> 画布无界，思考有方。拆解、关联、汇聚，打破边界连节点，汇聚思路成蓝图。

> **注意：** 此版本为基于原作 [famotime/siyuan-canvas](https://github.com/famotime/siyuan-canvas) 的二次开发分支，增加了多项功能增强与 UI 优化。

## 楔子

其实我不常用画布功能，只不过手头有一些别人分享的 Obsidian Canvas 文档，我的初衷是直接在思源笔记打开阅读，而不用重新回到 Obsidian——还是对“All-in-One”有执念，既然已经大费周章的迁移到思源笔记，我就希望所有的笔记都尽量汇合在一处，所有操作都尽量在一个工作台完成，不想成为来回切换的多持用户。

我为此研究了一下，发现 Obsidian 已经把 .canvas 文件格式开放了（[obsidianmd/jsoncanvas: An open file format for infinite canvas data.](https://github.com/obsidianmd/jsoncanvas)）。太好了，这样解析文件，兼容互通就方便多了。于是，大干几天之后，“无界 siyuan-canvas”插件诞生了。

## 核心功能与使用场景

在思源笔记中打开、编辑和导出 Obsidian `.canvas` 画布文件。基于开放的 JSON Canvas 格式，实现两个笔记工具之间的画布互通。我们为你打造了沉浸式的工作流，而且 **画布默认自动保存**，让你专注思考，不被打断。

**知识梳理** — 把零散的笔记和资料放到一张画布上，用连线建立关联，用分组按主题归类，形成可视化的知识网络。

**思路演示与汇报** — **（全新特性）** 利用“演示模式”录制卡片的播放路径。像幻灯片一样，一步步压暗背景并高亮展示当前节点，让网状的知识图谱也能以线性的逻辑向他人讲述。

**项目规划** — 用文本卡片列出任务，用连线表示依赖关系，用分组划分阶段，用颜色标记优先级，拖拽排版快速调整布局。

**文件管理** — 通过右侧边栏「无界文件」面板集中管理工作区所有画布，新建、重命名、复制、删除、打开所在目录，无需离开工作台。

**资料收集** — 从浏览器复制图片直接粘贴到画布，搜索思源中的文档和图片素材添加为卡片，所有参考资料集中在一个画面上。

**随时随地查阅** — **（全新特性）** 完美适配移动端仅查看模式，在手机上双指捏合轻松缩放，随时随地回顾你的思维全景。

**侧边栏文件管理（全新）** — **（全新特性）** 右侧边栏新增「无界文件」面板，无需打开标签页即可浏览工作区所有画布文件，支持新建画布/文件夹、按名称/时间排序、右键操作（重命名、复制、删除、打开所在目录），点击文件一键打开。

**跨工具协作** — 在 Obsidian 中创建的 `.canvas` 文件直接在思源中打开编辑，修改后保存回标准格式，两个工具间无缝流转。

### 强大的演示模式（路径录制）

不再只是静态的图谱，你可以让画布动起来：
- **路径录制**：选中节点添加到播放路径中，自由编排讲解顺序。
- **遮罩高亮**：播放时自动生成遮罩背景，高亮当前节点，让观众聚焦于你正在讲解的部分。
- **灵活编辑**：支持在播放路径中途继续录制或回退截断，随时调整演示逻辑。

### 个性化与视觉体验

- **全新主题配色**：内置多种精心调配的颜色主题，包括冷调彩虹、赛博霓虹等，让你的画布不再单调。
- **极简纯卡片（全新默认）**：卡片默认不显示标题栏，直接展示纯粹内容，减少视觉干扰。拖拽卡片任意位置即可移动。如需标题栏可在设置中重新开启。
- **卡片一行尺寸**：新建文本/文件/链接卡片默认 250×50px（一行高度），最小 50×50px，保持画布紧凑整洁。
- **锚点完整可见**：连线锚点圆完整显示在卡片边缘外侧，不再被裁剪为半圆。

### 四种卡片类型

|卡片|用途|举例|
| ----| --------------------------------| --------------------------------------|
|**文本**|写下想法、笔记、备注|记录会议要点、写下灵感碎片（双击新建默认文本）|
|**文件**|关联思源中的文档、图片或其他画布|把相关笔记卡片拖到画布上，形成知识地图|
|**链接**|放置网页地址|参考资料、外部文档的入口|
|**分组**|将相关卡片归类分组|按主题、阶段或优先级分组|

- 文本卡片支持 Markdown 渲染，标题、列表、代码块、引用等都能直接预览
- 文件卡片能识别思源文档、块 ID、图片素材和嵌套画布，双击即可跳转到对应内容
- 从剪贴板粘贴的图片会自动保存为文件卡片

### 连线与关系表达

- 从卡片四边的锚点拖拽即可创建连线，释放到空白区域弹出菜单选择卡片类型
- **添加文本卡片**：直接创建文本卡片并连线
- **添加笔记卡片（全新）**：打开文件选择器选择思源文档，选中后自动创建文件卡片并连线
- 连线支持添加文字标签，标注关系类型（如"导致"、"属于"、"参考"）
- 可设置箭头方向（无箭头、单向、双向）和颜色
- **点击连线靠近端点（全新）**：直接启动端点拖拽重连，无需先选中连线
- 拖拽端点可重新连接到其他卡片，或释放到空白区域弹出菜单创建新卡片
- **贝塞尔曲线优化**：采用 Euclidean 距离 + 0.5 因子计算控制点，连线弧度更自然流畅

![image](https://raw.githubusercontent.com/famotime/siyuan-canvas/main/assets/image-20260501191812-6nsesw1.png)

### 排版与布局

选中多个卡片后，浮动工具栏提供多种排版操作：

- **对齐** — 左 / 水平居中 / 右 / 上 / 垂直居中 / 下
- **排列** — 横排、纵排、网格（自动计算列数）
- **等距分布** — 水平 / 垂直方向均匀分布
- **拉伸对齐** — 水平 / 垂直方向铺满选区

![image](https://raw.githubusercontent.com/famotime/siyuan-canvas/main/assets/image-20260501190246-cbut0pi.png)

### 右侧属性面板与侧边栏

- **右侧属性面板**：选中卡片或连线后，面板显示「选区」标签页，可编辑位置、尺寸、颜色、文本内容、连线锚点方向和标签等属性。原有的文档浏览功能已迁移至右侧边栏。
- **右侧边栏文件浏览器（全新）**：点击右侧边栏「无界文件」图标，可快速浏览工作区中所有画布文件。支持新建画布/文件夹、按名称/编辑时间排序（保存/重命名后自动刷新排序）、右键菜单（重命名、复制、复制路径、删除、打开所在目录），点击文件即可在标签页中打开。文件变更（新建、保存、重命名、删除）时自动刷新列表。
- **打开所在目录（全新）**：在文件树或侧边栏的右键菜单中，可快速打开画布文件所在的系统文件夹（支持 Windows、macOS、Linux）

### 当前交互模型

- 从顶栏、命令面板或右侧边栏文件浏览器打开空白画布。
- 使用顶部工具栏新建画布、导入本地 `.canvas` 文件或「另存为」。
- **右侧边栏文件浏览器（全新）**：浏览工作区画布文件，新建/复制/重命名/删除，按名称或编辑时间排序并自动刷新，右键菜单可打开所在目录。
- 激活画布表面以显示底部工具栏，进行添加文本、文件、连线和分组等操作。
- 拖拽卡片任意位置即可移动（标题栏默认隐藏；可在设置中重新开启）。
- 按住 `Shift` 拖拽约束为仅垂直/水平方向移动；按住 `Alt`/`Option` 拖拽复制出一张相同卡片。
- 拖拽卡片边缘手柄或右下角手柄调整大小。
- 工具栏点击网格对齐按钮，拖拽/调整大小时自动吸附到 20px 网格。
- **智能对齐参考线**：拖拽卡片时，与其他卡片边缘或中心对齐时自动显示参考线并吸附。
- **从卡片锚点拖拽**创建新连线——释放到另一张卡片上完成连接，或释放到空白区域弹出菜单（添加文本卡片 / 添加笔记卡片）。
- **点击连线端点附近**启动端点拖拽重连。
- **添加笔记卡片**：打开思源文件选择器；选中文档后自动创建文件卡片并连线。
- 在右侧属性面板的「选区」标签页中编辑节点和连线属性（文档浏览已迁移至侧边栏）。
- 双击链接节点打开 URL。
- 双击 `.canvas` 文件节点在新的插件标签页中打开。
- 使用创建连线对话框或浮动连线工具栏创建和调整连接。
- 快速预览：使用 `Ctrl+Shift+Alt+C` 在当前文档中插入 Canvas 预览弹窗。

### 项目结构

- `src/index.ts` 保持插件生命周期精简，将运行时检测、标签页接线、设置 UI 和 Canvas 嵌入命令执行委托给 `src/canvas/` 中的专用辅助模块。
- `src/canvas/use-canvas-editor.ts` 是编辑器组合入口，文件操作、文件节点元数据、指针手势、选区工具栏 UI 和工作区树辅助功能分别拆分到独立模块。
- `src/components/canvas/CanvasWorkspace.vue` 作为主工作区组合视图，配合 `CanvasFileCard.vue`、`CanvasCreateEdgeDialog.vue` 和工作区上下文菜单辅助组件，从大型 SFC 中提取出聚焦的 UI 行为。
- `src/canvas/workspace-tree-core.ts` 和 `src/canvas/canvas-embed-command.ts` 将纯工作区树逻辑和嵌入命令编排隔离在可测试的依赖注入 API 之后。
- `src/canvas/file-target-resolution.ts`、`src/canvas/file-target-preview.ts` 和 `src/canvas/file-preview-fallbacks.ts` 组成主要的文件预览管线，旧的 `file-node-*` 模块为兼容适配器。
- `src/canvas/siyuan-file-node-lookups.ts` 包含思源文档和资源解析的纯查询逻辑，运行时 SQL 访问隔离在 `src/canvas/siyuan-kernel-file-node-lookups.ts` 中。
- `tests/` 与 canvas 模块对应，覆盖编辑器流程、插件生命周期、主题同步、显示辅助、解析、几何和持久化等方面的 Vitest 测试。

### 说明

- 底层遵循 `obsidianmd/jsoncanvas` 开放规范
- 保存时会保留规范未定义的额外字段，确保与 Obsidian 兼容互通
- 当前版本优先保证文件格式兼容和核心编辑体验，不追求与 Obsidian Canvas 的完全视觉一致
